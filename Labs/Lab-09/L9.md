# Lab 9: Source Credibility Analyzer Agent

## Overview

This week we're learning about **agents**, which are LLM systems that use tools in a loop in order to accomplish a goal autonomously. Instead of the developer scripting every step, an agent is given tools and instructions, and it figures out the path on its own.

In this lab, you will build a **Research Source Credibility Analyzer Agent** using the [OpenAI Agents SDK](https://openai.github.io/openai-agents-python/).

In your upcoming [Research Report](../../research/report.md), you are asked to work with _"credible sources"_, but what does this mean? Given a URL, your agent will read the source, investigate the author and publication, verify key claims against other sources, and produce a structured credibility report. This is both directly useful for your research project and will also teach you how agents plan, act, and reason across many steps.

### Concepts

1. **Agent Loop:** Letting the LLM decide what to do next, repeatedly, until a task is complete.
2. **Tool Design:** Building tools that give the agent real capabilities (reading, searching, writing).
3. **The "Think" Tool Pattern:** Using a structured tool with no side effects to force the agent to reason carefully before producing output.
4. **System Prompting for Agents:** Writing instructions that guide multi-step investigative behavior, including how to handle missing information.

## 2. Background: How Do You Evaluate Source Credibility?

Before you can build an agent that evaluates sources, you need to understand what credibility evaluation actually involves. When a researcher evaluates a source, they typically consider:

1. **Authorship:** Who wrote this? What are their qualifications? Are they an expert on this topic, or a random person with a blog? Can you even find out who wrote it?
2. **Publication:** Where was this published? Is it a peer-reviewed journal, a major news outlet (e.g., Reuters, CBC, AP), a government site (e.g., `.gc.ca`, `.gov`), or a personal blog? Does the publication have an editorial review process?
3. **Evidence and Citations:** Does the article cite its sources? Can you follow those citations to real, primary sources? Or does it make claims without backing them up?
4. **Corroboration:** Are other credible sources reporting the same thing? If a claim only appears on one obscure website, that's a potential red flag.
5. **Bias and Tone:** Is the language neutral and informational, or emotional and persuasive? Does the piece acknowledge counterarguments, or does it present only one side?
6. **Currency:** When was this published? Is the information still current, or has it been superseded by newer research or events?

Your agent needs to investigate all of these dimensions. It can't just read the page and say _"this looks credible!"_

## 3. Step-by-Step Instructions

After you have researched, configured, and setup your Agents SDK, focus on using it to define your tools.

### Step 1: Implement the `read_url` Tool

Your agent needs to be able to read web pages. Use the [Jina Reader API](https://jina.ai/reader/) to fetch a URL and return its content as Markdown. This is the same approach used in the research agent example from the lecture notes.

The agent will use this tool to:

- Read the source being evaluated
- Read "About" pages, author bios, and editorial policies on the same site
- Read corroborating or contradicting articles it discovers during research

```python
# Use the requests library or urllib
import requests

def read_url(url: str) -> str:
    # TODO: add error handling...
    response = requests.get(f"https://r.jina.ai/{url}")
    text = response.text
    # Truncate to avoid blowing up the context window
    return text[:10000]
```

```ts
async function readUrl(url: string): Promise<string> {
  // TODO: add error handling...
  const response = await fetch(`https://r.jina.ai/${url}`);
  const text = await response.text();
  // Truncate to avoid blowing up the context window
  return text.slice(0, 10000);
}
```

> [!TIP]
> You may need to experiment with the truncation limit. Too short and the agent misses important content at the bottom of articles (like author bios or references). Too long and you burn through your context window. Start with `10,000` characters and adjust based on what you observe.

### Step 3: Implement the `evaluate_credibility` Tool

This is the most important and most unusual tool in your agent. It **doesn't do anything**! Instead, we'll use it to **force the agent to stop and organize its thinking** into a structured evaluation before it produces its final report.

This is an application of the **"think" tool pattern** described in [Anthropic's research](https://www.anthropic.com/engineering/claude-think-tool). The idea is that agents perform better when they have a dedicated space to reason carefully, especially before making a final judgment. By making this a tool with a detailed schema, we force the agent to fill out every field, which prevents it from skipping steps or making up its assessment.

The tool's schema **is** the credibility rubric. Here's a suggested structure you should implement in your schema (feel free to modify and improve it):

| Field                                            | Type    | Description                                                                                                                                                               |
| ------------------------------------------------ | ------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `source_url`                                     | string  | The URL being evaluated                                                                                                                                                   |
| `source_type`                                    | enum    | One of: `peer_reviewed_journal`, `news_organization`, `government_agency`, `nonprofit_organization`, `corporate_blog`, `personal_blog`, `social_media`, `wiki`, `unknown` |
| `author.name`                                    | string  | Author name. If unknown after investigation, write `"Unknown"` and explain what you checked                                                                               |
| `author.credentials`                             | string  | What qualifications does the author have? If you couldn't find them, describe what you searched for                                                                       |
| `author.credibility_assessment`                  | string  | Your assessment of author credibility. If unknown, explain how this affects your overall assessment                                                                       |
| `publication.name`                               | string  | Publication or website name. If not a recognized outlet, use the domain name                                                                                              |
| `publication.reputation`                         | string  | What did your research reveal about this publication? If unknown, describe what you searched for                                                                          |
| `publication.editorial_process`                  | enum    | One of: `peer_reviewed`, `editor_reviewed`, `self_published`, `unknown`                                                                                                   |
| `content_analysis.claims_supported_by_evidence`  | boolean | Are the main claims backed by data, citations, or primary sources?                                                                                                        |
| `content_analysis.sources_cited`                 | boolean | Does the article cite its sources?                                                                                                                                        |
| `content_analysis.corroborated_by_other_sources` | boolean | Did you find other credible sources reporting the same claims?                                                                                                            |
| `content_analysis.contradicted_by_other_sources` | boolean | Did you find credible sources that contradict the claims?                                                                                                                 |
| `content_analysis.language_is_objective`         | boolean | Is the tone neutral and factual, or emotional and persuasive?                                                                                                             |
| `content_analysis.date_published`                | string  | When was this published? Is it current?                                                                                                                                   |
| `overall_credibility`                            | enum    | One of: `high`, `medium`, `low`, `very_low`                                                                                                                               |
| `reasoning`                                      | string  | Explain your overall credibility rating, citing specific evidence from your research                                                                                      |

The tool's implementation function simply returns the evaluation data back unaltered. It doesn't write to a database or call an API. Instead, it should ensure the structured assessment appears in the agent's context:

```python
# Python hint
def evaluate_credibility(evaluation: dict) -> dict:
    return {
        "status": "evaluation_recorded",
        "evaluation": evaluation
    }
```

```ts
// TypeScript hint
async function evaluateCredibility(evaluation: object): Promise<object> {
  return {
    status: 'evaluation_recorded',
    evaluation,
  };
}
```

By returning the analysis data back to be included in context as a tool result, the agent can reference its own structured evaluation when writing the final report.

> [!IMPORTANT]
> The power of this tool is in the **schema**, not the implementation. The schema forces the agent to produce a structured, complete assessment. Spend time getting the schema right. Use Pydantic (Python) or Zod (JS/TS) to define it precisely, with good `.describe()` annotations on every field. The descriptions guide the agent on what to put in each field.

### Step 4: Implement the `write_file` Tool

Once the agent has completed its investigation and evaluation, it needs to write a final credibility report to a Markdown file. This tool takes a filename and Markdown content string, and writes the file to disk.

### Step 5: Write the System Prompt

We need to teach the agent **how to investigate**. The system prompt is one of the most important parts of your agent, since it determines whether the agent does a shallow analysis or genuine investigative work.

Your system prompt should instruct the agent to follow a structured process. Here is a suggested framework to build from, and you should use your own words to expand on it and write it out yourself:

1. **Read** the source carefully. Identify the main claims, the author, the publication, and the date.
2. **Investigate the author.** Search for their name, credentials, affiliations, and other work. Are they an expert on this topic?
3. **Investigate the publication.** Search for information about the outlet. Is it a known, reputable source? Does it have an editorial review process? Check for an "About" page on the same domain.
4. **Verify the claims.** Search for the key claims made in the article. Can you find other credible sources reporting the same thing? Are there fact-checks or contradictions?
5. **Check for bias indicators.** Is the language neutral or emotionally charged? Is the piece trying to inform or persuade? Are counterarguments acknowledged?
6. **Evaluate.** Use the `evaluate_credibility` tool to formally record your structured assessment. Every field must be based on evidence you actually found — do not guess.
7. **Report.** Write a final Markdown credibility report using the `write_file` tool. The report should summarize your findings, include the structured evaluation, and provide a clear verdict with reasoning.

Remember that an agent needs to follow a series of steps, use tools, and achieve a goal. Your system prompt should help it understand all this.

#### Handling Missing Information

Not every source will have clear authorship or publication information. Your system prompt **must** include instructions for what the agent should do when it hits dead ends. Without these instructions, the agent will either hallucinate information or give up. Consider including guidance like:

- **If the author cannot be identified:** Check for "About," "Team," or "Contact" pages on the same domain. Search the web for the article title in quotes to see if it's attributed elsewhere. If you still can't find the author, record this as a credibility concern — anonymous or unattributed sources are inherently less verifiable.
- **If the publication is unknown:** Look for an "About" page on the same domain. Search for the domain name to learn about the outlet. Check if the site has an editorial policy or masthead. If the publication is unknown or unestablished, note this. It doesn't automatically mean the content is wrong, but it means you must rely more heavily on corroboration of the claims.
- **If claims cannot be verified:** Note specifically which claims you could and could not verify. Distinguish between "I found contradicting evidence" and "I found no evidence either way." These are very different situations.
- **General principle:** Never fabricate information to fill gaps. If you don't know something after investigating, say so explicitly. An honest "I could not determine..." is always better than a guess.

You should also deal with cases where the URL can't be downloaded (e.g., `403` or `404` errors), rate limits on web searches, etc. Make sure your agent knows what to do when things go wrong.

## 4. Testing

Test your agent against the following types of sources. You should find **one real URL** for each category and run your agent against it. Choose sources relevant to your research project topic if possible.

1. **High credibility — Government or institutional source:** A page from a government website (e.g., `canada.ca`, `statcan.gc.ca`, a US `.gov` site) or an established institution (e.g., a university, WHO, a national research council).

2. **High credibility — Major news organization:** An article from a well-known news outlet with editorial standards (e.g., Reuters, AP, CBC, The Globe and Mail, The New York Times, BBC).

3. **Medium credibility — Organizational or corporate blog:** A post from a company blog, industry publication, or advocacy organization. These may have useful information but also have a perspective or agenda.

4. **Low credibility — Personal blog or opinion site:** A post from an unknown author on a personal blog, Medium, Substack, or similar platform with no editorial oversight.

5. **Tricky — A source that looks credible but has issues:** This is the most interesting test. Find a source that _appears_ authoritative but has a problem: it might be outdated, from a predatory journal, retracted, heavily biased while appearing neutral, or from a site that mimics the style of a credible outlet. This tests whether your agent does genuine investigation or just pattern-matches on surface appearance.

> [!TIP]
> If you're struggling to find a "tricky" source, try searching for articles on controversial health or science topics, which are full of professional-looking sites that don't hold up under scrutiny.
