## Review: Data URIs

A Data URI encodes a file's content directly into a URL string. The format is:

```text
data:<media-type>;base64,<encoded-data>
```

For example, a tiny PNG image as a Data URI:

```
data:image/png;base64,iVBORw0KGgoAAAANSUhEUgAAAAEAAAABCAYAAAAfFcSJAAAADUlEQVR42mNk+M9QDwADhgGAWjR9awAAAABJRU5ErkJggg==
```

The three components are:

| Component          | Example          | Purpose                                      |
| ------------------ | ---------------- | -------------------------------------------- |
| **Scheme**         | `data:`          | Identifies this as a Data URI                |
| **MIME type**      | `image/png`      | Tells the consumer what kind of data this is |
| **Base64 payload** | `iVBORw0KGgo...` | The file's binary content encoded as text    |

Your library will handle the following media types:

| Category   | MIME Types                                                            |
| ---------- | --------------------------------------------------------------------- |
| **Images** | `image/png`, `image/jpeg`, `image/gif`, `image/webp`, `image/svg+xml` |
| **Audio**  | `audio/mpeg` (MP3), `audio/wav`, `audio/ogg`                          |
| **Video**  | `video/mp4`, `video/webm`                                             |

> [!NOTE]
> SVG is a special case — it's XML text, not binary. Your library should handle both binary and text-based media correctly.

/**
 * The three supported media categories.
 */
export type MediaCategory = 'image' | 'audio' | 'video';

/**
 * A parsed Data URI with its components separated out.
 */
export const DataURISchema = z.object({
  /** The MIME type (e.g. "image/png") */
  mediaType: MediaTypeSchema,
  /** The media category: image, audio, or video */
  category: z.enum(['image', 'audio', 'video']),
  /** The raw Base64 string (without the data:...;base64, prefix) */
  base64: z.string().min(1),
  /** The complete Data URI string */
  raw: z.string().startsWith('data:'),
});

export type DataURI = z.infer<typeof DataURISchema>;

/**
 * Maps file extensions (without the dot) to MIME types.
 */
export const EXTENSION_TO_MIME: Record<string, MediaType> = {
  png: 'image/png',
  /** Allow both .jpg and .jpeg */
  jpg: 'image/jpeg',
  jpeg: 'image/jpeg',
  gif: 'image/gif',
  webp: 'image/webp',
  svg: 'image/svg+xml',
  mp3: 'audio/mpeg',
  wav: 'audio/wav',
  ogg: 'audio/ogg',
  mp4: 'video/mp4',
  webm: 'video/webm',
};

#### Python - `src/types.py`

```python
# src/types.py
from enum import Enum
from pydantic import BaseModel, field_validator


class MediaType(str, Enum):
    """All MIME types supported by this library."""
    PNG = "image/png"
    JPEG = "image/jpeg"
    GIF = "image/gif"
    WEBP = "image/webp"
    SVG = "image/svg+xml"
    MP3 = "audio/mpeg"
    WAV = "audio/wav"
    OGG = "audio/ogg"
    MP4 = "video/mp4"
    WEBM = "video/webm"


class MediaCategory(str, Enum):
    """The three supported media categories."""
    IMAGE = "image"
    AUDIO = "audio"
    VIDEO = "video"


class DataURI(BaseModel):
    """A parsed Data URI with its components separated."""
    media_type: MediaType
    category: MediaCategory
    base64: str
    raw: str

    @field_validator("base64")
    @classmethod
    def base64_not_empty(cls, v: str) -> str:
        if not v.strip():
            raise ValueError("base64 must not be empty")
        return v

    @field_validator("raw")
    @classmethod
    def raw_starts_with_data(cls, v: str) -> str:
        if not v.startswith("data:"):
            raise ValueError("raw must start with 'data:'")
        return v


# Maps file extensions (without dot) to MIME types
EXTENSION_TO_MIME: dict[str, MediaType] = {
    "png": MediaType.PNG,
    # Allow both .jpg and .jpeg
    "jpg": MediaType.JPEG,
    "jpeg": MediaType.JPEG,
    "gif": MediaType.GIF,
    "webp": MediaType.WEBP,
    "svg": MediaType.SVG,
    "mp3": MediaType.MP3,
    "wav": MediaType.WAV,
    "ogg": MediaType.OGG,
    "mp4": MediaType.MP4,
    "webm": MediaType.WEBM,
}


def get_category(mime_type: MediaType) -> MediaCategory:
    """
    Extract the category from a MIME type.
    e.g. "image/png" -> MediaCategory.IMAGE
    """
    prefix = mime_type.value.split("/")[0]
    return MediaCategory(prefix)
```

## Part 2: Red Phase: Write Failing Tests

Now you will write tests **before any implementation exists**. This is the most important part of the lab. Your tests become the specification that the agent must satisfy, and how it will verify its own work.

### Step 1: Plan Your Test Cases

Before writing any code, think about what your library needs to handle. Here are the categories of behavior to test:

**Encoding (file → Data URI):**

- Encode a PNG file and verify the Data URI starts with `data:image/png;base64,`
- Encode a JPEG and verify correct MIME type
- Encode an SVG file (text-based, not binary)
- Encode an MP3 file and verify correct MIME type
- Verify that encoding then decoding produces the original bytes (round-trip)
- Error: file doesn't exist → should throw/raise
- Error: unsupported file extension (e.g., `.bmp`) → should throw/raise
- Error: empty file → should throw/raise

**Decoding (Data URI → buffer/bytes):**

- Decode a known Data URI and verify the bytes match the original file
- Parse the MIME type correctly from a Data URI string
- Parse the Base64 payload correctly
- Error: malformed Data URI (missing `data:` prefix) → should throw/raise
- Error: malformed Data URI (missing `;base64,`) → should throw/raise
- Error: unsupported MIME type in the URI → should throw/raise
- Error: invalid Base64 content → should throw/raise

### Step 3: Monitor the ReAct Loop

As the agent works, observe the cycle:

1. Agent writes code
2. Agent runs tests
3. Some tests fail
4. Agent reads the error output
5. Agent edits the code
6. Agent runs tests again
7. Repeat until green

**This is "closing the loop" in action.** The test results are the agent's grounding mechanism, which prevent it from hallucinating progress.

### Step 4: Steer as Needed

While the loop is running, if the agent goes off track, provide guidance as soon as you see it happen:

```text

```

```text

Include the actual file extension in the error so the
user knows what went wrong.
```

Codex will incorporate your additional instructions into the loop on the next iteration (i.e., you don't have to wait for it to finish).
